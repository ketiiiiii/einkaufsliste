## 1. Bugfixes

### Build-Fehler: Doppeltes `}` beim Board-Kommentar
- **Problem:** `{/* Board view */}}` — extra `}` verursachte Parsing-Fehler in Next.js Turbopack
- **Fix:** Überflüssige geschweifte Klammer entfernt

### Hydration-Mismatch: `toLocaleString("de-CH")`
- **Problem:** `Math.round(n).toLocaleString("de-CH")` liefert auf Server (Node.js) und Browser unterschiedliche Ausgaben → React Hydration-Fehler
- **Fix:** Manuelle Formatierung mit Regex (`replace(/\B(?=(\d{3})+(?!\d))/g, "\u2019")`) — Apostroph als CH-Tausendertrennzeichen, kein locale-API

## 2. Task Board — neue Features

### Durchlaufzeiten (AT)
- Jede Task-Karte hat unten links ein `<input type="number">` Feld (`1d` Standard)
- Wert wird in `localStorage` gespeichert
- Wird für CPM-Berechnung und Preisanzeige verwendet

### Auto-Layout
- Button **"⬡ Auto-Layout"** in der Toolbar (nur aktiv wenn Verbindungen vorhanden)
- Algorithmus: **Kahns topologisches Sortieren** → Spalten/Zeilen-Zuordnung
- Taskkarten werden automatisch von links nach rechts nach Prozessfluss angeordnet

### Kritischer Weg (CPM — Critical Path Method)
- Algorithmus: **Vorwärtspass** (ES/EF) + **Rückwärtspass** (LF/LS) + Float-Berechnung
- Kritisch = Float < 0.001
- **Visuelle Darstellung:**
  - Orangener Ring auf kritischen Task-Karten (`ring-2 ring-orange-400`)
  - Orangene Pfeile für kritische Verbindungen (eigener SVG-Marker `#arrowhead-critical`)
  - Orangener Info-Strip: *"Projektdauer: X Tage — Orangene Knoten & Kanten = kein Zeitpuffer"*
- **Hinweis:** Nur sichtbar wenn mind. 1 Verbindung vorhanden

### Preisanzeige
- **Pro Karte:** CHF-Betrag unten rechts (Tage × 8h × CHF 220)
- **Gesamt-Strip:** `X Tage × 8h × CHF 220 = CHF XXX` unterhalb der Toolbar
- Stundensatz: CHF 220 / AT = 8h

### Tabellenansicht — Erweiterung
- Neue **"Tage"-Spalte** mit inline Number-Input
- **Kritischer-Pfad-Marker**: Orangener Dot (`●`) in der Titel-Spalte
- `colSpan` von 5 auf 6 erhöht

### Todos & Kommentare pro Task (Detail-Modal)
- Öffnen via **✎-Button** oben rechts auf jeder Karte
- **Todos/Besorgungen:**
  - Checkbox-Liste mit Durchstreichen bei Erledigung
  - Hinzufügen / Löschen
- **Kommentare:**
  - Textarea mit Timestamp
  - **Screenshot einfügen via Ctrl+V** → Bild aus Zwischenablage wird direkt als Base64 eingefügt und angezeigt
  - Bildvorschau vor dem Senden
  - Kommentare löschen
- Alles wird in `localStorage` gespeichert

## 3. Produkt-Composer

### Konzept
Ermöglicht es, mehrere Watsoft-Produkte zu einem kombinierten Projektboard zusammenzustellen. Jedes Produkt hat eigene Phasen/Tasks und optionale Module. Werden mehrere Produkte gewählt, werden automatisch Cross-Produkt-Verbindungen gesetzt.

### Implementierte Produkte (`src/lib/product-library.ts`)

| Produkt | Farbe | Phasen | Optionen |
|---|---|---|---|
| **Wato Kasse** | amber | Kickoff → HW → Install → Stammdaten → Config → Schulung → Test → Go-Live | Kassenbuch, Webshop-Anbindung |
| **Wato DocCreator** | sky | Kickoff → Vorlagen → Datenquellen → Workflow → Test → Abnahme | E-Signatur, Archivierung |
| **proffix** | emerald | Kickoff → Install → Stammdaten → Fibu → Auftrag→ Schulung → Migration → Go-Live | Lohnmodul, Webshop ERP-Anbindung |
| **pxApp** | violet | Analyse → Config → Rechte → Masken → Test → Rollout | Zeiterfassung, Offline-Modus |

### Cross-Produkt-Verbindungen (automatisch)
- `proffix Go-Live` → `pxApp konfigurieren` (App braucht laufendes ERP)
- `proffix Stammdaten` → `Wato Kasse Stammdaten import` (Kassen-Artikel aus ERP)
- `proffix Auftrag & Lager` → `DocCreator Datenquellen` (ERP liefert Dokument-Daten)
- `pxApp Benutzertest` → `Wato Kasse Abnahmetest` (Gesamttest erst wenn alles ready)

### Workflow (3-Schritt-Prozess)
1. **Produkte auswählen** — Kacheln mit Beschreibung, Task-Anzahl, Querverbindungs-Vorschau
2. **Module konfigurieren** — Checkboxen pro Produkt (inkl. +AT Anzeige)
3. **Vorschau** — Mini-Board mit skalierten Karten + Pfeilen, Gesamtsumme AT → "Auf Board laden"

### Neue Dateien
- `src/lib/product-library.ts` — Produktdefinitionen, Board-Generator
- `src/app/tasks/ProductComposer.tsx` — 3-Schritt UI-Komponente

### Integration
- Neuer Button **"⊞ Produkte kombinieren"** in `TasksPageClient.tsx` neben "✦ Projektablauf generieren"
